**Project Title:** Centralised Project Dashboard Development  
**Project Objective:** To develop a single, simple, and easy-to-navigate platform that aggregates links and access points for all Neodym internal tools and client projects. The dashboard will serve as a portfolio and an internal hub, allowing users to find and directly access projects without searching for individual credentials or links.  
**Core Requirements:**

* **Categorization & Structure:** The dashboard must visually separate projects into two primary categories: "Internal Projects" (Neodym) and "Client Projects" (e.g., Triangle IP).  
* **Scalability:** The system must feature a tree-like structure that allows users to easily add new clients or append new projects under existing internal or client categories as the company scales.  
* **User Interface (UI):** The UI must be intuitive, user-friendly, and provide a decent experience. Clicking on a category (like Neodym) should display all associated projects—such as the "token tracker project" or "recruitment drive project"—with direct links to the tools.  
* **Access Control & Authentication:** The ability to add or remove projects from the dashboard should be accessible to anyone logged in with the standardized `hello@neodym.ai` account. Furthermore, the project aims to standardize the login credentials across all specific tools, utilizing the `hello@neodym.ai` email and one highly secure, universal password to eliminate the need to search for auto-generated passwords.  
* **Pre-Authorized Project Access:** Project links added to the dashboard must already be authorized with the required credentials wherever technically possible. The intended user experience is that a team member clicks a project link from the dashboard and the project opens directly without requiring a separate login, credential lookup, or manual authentication step.  
* **Hosting:** The platform does not require heavy computational resources. The first preference is to host it on Neodym's paid Vercel account to prevent latency issues, provided there is room. If not, a free hosting alternative like Railway is acceptable.  
* **Information Gathering:** The developer must collaborate directly with team members Ibrahim, Asher, and Hamza to collect all relevant website links and login credentials for the projects they have worked on
